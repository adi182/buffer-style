Inputs

Inputs are molecules that capture content entered by the user. They have a label atom and an input atom. Labels should always be present when using an input for accessibility.

Input

Use this molecule to capture content entered by the user and communicate the intention of what data will be captured by the input atom with the label atom.

Example: 
<label class="input-label" for="email">Email</label>
<input class="input" id="email" placeholder="email@domain.com" type="text">

Input with a left-aligned icon

Use this molecule in instances where an icon communicates intention with the user.

Example: 
<div class="input-with-icon">
  <label class="input-label" for="search-profiles">Search Profiles</label>
  <div class="input-with-icon-left">
    <i class="bi bi-search"></i><input class="input input--icon-left" id="search-profiles" placeholder="@buffer" type="text">
  </div>
</div>

Input with a right-aligned icon

Use this molecule in instances where an icon communicates intention with the user.

Example: 
<div class="input-with-icon">
  <label class="input-label" for="twitter-username">Twitter Username</label>
  <div class="input-with-icon-right">
    <i class="bi bi-circle-twitter"></i><input class="input input--icon-right" id="twitter-username" placeholder="@buffer" type="text">
  </div>
</div>
